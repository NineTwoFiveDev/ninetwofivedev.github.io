## Narrative Reflection

The artifact I chose for the Software Design and Engineering category is a data management and validation application. I originally created it during my CS 320: Software Testing, Automation, and Quality Assurance class. In its initial form, the project was written in Java and used JUnit testing for basic unit tests on simple data objects. It aimed to ensure that in-memory data structures met functional requirements without failing or throwing unhandled exceptions. I picked this artifact because CS 320 marked a turning point in my academic career. It was the course where I discovered a genuine passion for software engineering, especially for detail and quality assurance. Including this piece in my ePortfolio shows my understanding of software reliability.

To raise this project to a capstone level, I completely restructured and expanded it. First, I translated the entire codebase from Java to C#. Next, I increased the complexity of the data models and used C#’s improved getters and setters to ensure strict object-level data validation. Finally, to support these structural changes, I rewrote and expanded the unit-testing suite to cover all new data fields and edge cases. These efforts demonstrate my ability to work with different technology stacks and my commitment to secure, test-driven development.

Through these improvements, I successfully met the course outcomes I targeted in Module One, with no major updates needed for my coverage plan.

Outcome 3 (Design and evaluate computing solutions): Porting the application involved assessing the original Java version and redesigning it to fit C# and .NET standards while managing the structural trade-offs between the two languages.

Outcome 4 (Use innovative techniques, skills, and tools): Transitioning to a new language ecosystem enabled me to leverage modern C# features and testing frameworks, resulting in a highly functional, industry-standard solution.

Outcome 5 (Develop a security mindset): By implementing getters and setters in the updated data models, I safeguarded the application's internal data from unauthorized access. This shows a proactive approach to software security and data integrity.

I found the process of modifying this artifact incredibly rewarding, although it had its challenges. Translating logic from Java to C# deepened my understanding of how different languages and compilers handle memory, object-oriented principles, and syntax. I realized that porting code is rarely a straightforward translation; it requires a solid grasp of the best practices for both languages.

One major challenge was ensuring my new unit tests accurately reflected the original intent of the Java tests while accommodating the increased data complexity. Debugging failing tests during the transition forced me to analyze data logic more thoroughly than I had in the original project. Ultimately, overcoming these challenges strengthened my critical thinking and logical reasoning skills, demonstrating that I can maintain high software quality standards even during a project migration to a new environment.
