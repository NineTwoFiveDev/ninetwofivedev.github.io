# Professional Self-Assessment

Finishing this computer science degree and putting together my ePortfolio has been a great way to reflect on how my coding has changed. When I first started, I aimed to write code that simply compiled and worked well. Now, I focus entirely on building clean, full-stack architectures that can scale without breaking. This capstone allowed me to tear apart my old projects and rebuild them using modern frameworks. I transformed clunky student code into optimized, secure software that fits well in a real production environment.

One key thought during this program is that software engineering isn’t just about typing code; it’s mostly about communication. You can create the most elegant code, but if it doesn't address the business problem or if your team can’t maintain it, it’s pointless. I have focused on learning to turn messy business requirements into clear code. I’ve worked through peer code reviews and explained technical limitations to non-technical stakeholders without getting bogged down in the details.

On the technical side, I have spent a lot of time breaking old habits related to data structures and algorithms. It’s easy to write nested loops that function well on small datasets, but they struggle under heavy loads. During my project upgrades, I used Big-O notation to find performance bottlenecks. Replacing unoptimized arrays with efficient data structures significantly reduced the runtime. To me, development now means constantly weighing the trade-offs between memory use and execution speed.

That same approach applies to my approach to full-stack design and databases. I view applications as complete ecosystems in which the frontend, API endpoints, and backend storage must communicate smoothly. Whether I’m structuring schemas in MySQL or managing flexible data in MongoDB, I design with data integrity in mind. I want to ensure that the databases I create can grow with the application without requiring a major rewrite later on.

Security is another major focus in my workflow. It cannot be just a step you check off right before deployment; it must be integrated from the very first line of code. I have trained myself to look at my architecture from an attacker’s perspective to identify vulnerabilities early. In my current work, this means writing strict input validation to prevent injection attacks before they start, ensuring proper data encryption, and setting up strong access controls. Catching these issues during the initial design phase is essential for protecting data effectively.

The three artifacts following this assessment pull all these elements together. The first project emphasizes software engineering, showcasing a clean, modular architecture that is easy to read and maintain. The second project focuses on data structures and algorithms, where I eliminated slow logic pipelines to enhance speed. The final project integrates everything into a full-stack setup, connecting a functional user interface to a secure database backend. Together, these projects demonstrate where I am as a developer and my readiness to tackle complex software challenges.
